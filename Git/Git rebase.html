Git rebase

  • rebase和merge

    • merge 根据同一起点，把修改合并，然后生成一个commit
    • rebase 抽出分支的commit， 合并目标分支commit，置顶抽出的commit

  • merge

  • rebase

  • 在合并时，处理冲突：　

    • merge

    • rebase

  • git rebase -i HEAD~3 对所做的提交进行修改,选项：

  • git cherry-pick 取某一个分支中的一个或几个commit(s)来进行合并

  • git stash [pop/apply /drop/save] 将改动存入暂存区[弹出暂存区记录/恢复一个暂存区内记录]
  • git blame 文件逐行追溯
  • git clean 清除工作区未跟踪文件
